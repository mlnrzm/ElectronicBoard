установить docker контейнер
https://docs.docker.com/engine/install/ - оф руководство по установке
в папке с приложением с ИП сделать ./gradlew clean и ./gradlew build
далее в папке с приложением/build/libs/ забрать jar файл - самый большой по размеру нужен будет
этот файл положить в другую папку создать рядом Dockerfile со след содержимым

    FROM java:8
    VOLUME /tmp
    ADD project.jar app.jar
    RUN bash -c 'touch /app.jar'
    ENTRYPOINT ["java","-Djava.security.egd=file:/dev/./urandom","-jar","/app.jar"]
    
где указать НУЖНУЮ для конкретного проекта версию явы после двоеточия
project.jar - имя файла в папке, который будет добавлен в контейнер (переименовать при копировании из директории проекта)
    
далее в консоли
создаем образ:
sudo docker build -t docker_spring:latest ~/docker_spring/
   
после -t идет <имя_образа>:<версия (рекомендую просто писать latest)>
    
~/docker_spring - имя папки где лежит jar и Dockerfile

запускаем контейнер
ВАЖНО: если в системе остался systemd файл с 4 лабы - остановить службу иначе получим ошибку из-за уже занятого порта 8080 
sudo docker run -d -p 8080:8080 --name spring-app docker_spring
-d - запуск в фоновом режиме
-p 8080:8080 - соотношение портов контейнера и локальной машины
--name - имя контейнера (можно не указывать, задастся рандомное)
docker_spring - имя образа котрое указывали при создании

просмотр запущенных контейнеров
sudo docker ps
здесь же можно узнать всю инфу, как ID и имя запущенного контейнера

завершить контейнер
sudo docker kill spring-app
где spring-app - имя контейнера, либо автоматически созданное, либо указанное при запуске
